Des pages sans erreur de code
(page10)

Présentation

Un site de bonne qualité pour google est un site qui ne contient pas d'erreur de code et qui a passé la validation W3c.

Il y a des erreurs html sur la page contact, et également des erreurs CSS qui impacte l'affichage du site en responsive

Il est important de corriger toutes les erreurs car ces elements en erreur influent sur l'affichage du site ou sur le balisage html . Qui peut etre important pour le SEO

Conclusion

En principe un site sans erreur de code n'influera pas plus que les autres sur le référencement naturelle, mais il sera tout de meme considéré comme un site de qualité
